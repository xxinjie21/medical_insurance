## 🏗️ 系统架构概述

### 技术栈
- **后端框架**: Spring Boot 2.7.18
- **数据库**: MySQL 8.0.32
- **ORM 框架**: MyBatis-Plus 3.5.3.1
- **缓存**: Redis + StringRedisTemplate
- **权限控制**: JWT + 自定义拦截器
- **分布式锁**: Redisson

### 系统角色
| 角色 | 标识 | 说明 |
|------|------|------|
| 患者 | PATIENT(1) | 就诊、充值、报销 |
| 医院 | HOSPITAL(2) | 提供诊疗服务、录入费用、申报结算 |
| 医保局 | MEDICAL(3) | 审核批次、拨付基金 |

### 核心数据表
- **visit**: 就诊表
- **fee**: 费用明细表
- **settle**: 结算表
- **batch**: 申报批次表
- **batch_item**: 申报明细表
- **pay**: 基金拨付表
- **user_account**: 患者账户表
- **recharge_record**: 充值记录表
- **consumption_record**: 消费记录表

## 🔄 核心业务流程

### 1. 就诊结算流程

```
患者挂号 → 医院诊疗 → 添加费用明细 → 就诊结算 → 患者支付自付部分
```

**参与方**:
- 患者：提供就诊信息、支付自付费用
- 医院：录入费用、执行结算

**状态流转**:
```
就诊状态：待结算 (0) → 已结算 (1)
结算状态：待申报 (0) → 已申报 (1) → 已审核 (2) → 已拨付 (3)
```

### 2. 批次申报流程

```
医院创建批次 → 添加结算单 → 提交申报 → 医保局审核 → 基金拨付
```

**参与方**:
- 医院：创建批次、添加结算单
- 医保局：审核、拨付

**状态流转**:
```
批次状态：待申报 (0) → 已申报 (1) → 已完成 (2)
```

### 3. 账户充值支付流程

```
患者充值 → 生成充值记录 → 更新账户余额 → 就诊后使用余额支付自付部分
```

**参与方**:
- 患者：充值、支付
- 系统：记录充值、扣减余额

## 📡 接口调用详解

### 4.1 用户管理接口

#### 1. 用户注册接口

**接口**: `POST /user/sign`

**请求**:
```json
{
  "password": "123456",
  "name": "张三",
  "idCard": "360104199001011234",
  "role": 1
}
```

**响应**:
```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "id": 1,
    "name": "张三",
    "idCard": "360104199001011234",
    "role": 1,
    "createTime": "2026-04-27T10:30:00"
  }
}
```

**后端处理流程**:
```
1. 参数验证
   - 姓名、密码、角色不能为空
   - 患者必须传身份证，医院必须传医院 ID
   ↓
2. 密码 MD5 加密
   ↓
3. 构建用户对象
   ↓
4. 保存到数据库
   ↓
5. 返回用户信息
```

---

#### 2. 用户登录接口

**接口**: `POST /user/login`

**请求**:
```json
{
  "userId": 1,
  "password": "123456",
  "role": 1
}
```

**响应**:
```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "userId": 1,
    "name": "张三",
    "idCard": "360104199001011234",
    "role": 1,
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }
}
```

**后端处理流程**:
```
1. 接收请求（userId + password+role）
   ↓
2. 查询数据库验证用户
   ↓
3. 校验角色是否匹配
   ↓
4. 密码比对（MD5 加密后比较）
   ↓
5. 生成 JWT Token（包含 userId）
   ↓
6. 保存登录信息到 Redis
   - login:token:{token} → userId (2 小时)
   - login:role:{userId} → role (2 小时)
   ↓
7. 返回用户信息 + token
```

### 4.5 结算管理接口

#### 13. 就诊结算接口

**接口**: `POST /settle/calculate/{visitId}`

**权限**: 仅限医院角色 (HOSPITAL)

**请求**:
```
POST /settle/calculate/1
```

**响应**:
```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "id": 1,
    "visitId": 1,
    "totalAmount": 101.00,
    "reimburseAmount": 91.00,
    "selfPayAmount": 10.00,
    "status": 1
  }
}
```

**后端处理流程**:
```
1. 获取分布式锁
   - 锁 key: lock:settle:{visitId}
   - 防止重复结算
   ↓
2. 执行事务
   a. 幂等性检查
      - 查询是否已结算
      - 设置 Redis 幂等标识
   ↓
   b. 查询费用明细
      - 查询该就诊的所有费用
   ↓
   c. 计算报销金额
      - 甲类费用：100% 报销
      - 乙类费用：80% 报销
      - 自费费用：0% 报销
      - 先累加再四舍五入（避免精度误差）
   ↓
   d. 创建结算记录
      - totalAmount: 总费用
      - reimburseAmount: 报销金额
      - selfPayAmount: 自付金额
   ↓
   e. 更新就诊状态
      - status: 待结算 (0) → 已结算 (1)
   ↓
   f. 设置幂等性标识
      - idempotent:settle:visit:{visitId} (1 小时)
   ↓
3. 释放分布式锁
4. 返回结算信息
```

**金额计算示例**:
```
费用明细:
- 阿莫西林（甲类）: 51.00 × 100% = 51.00
- 血常规（乙类）: 50.00 × 80% = 40.00

总费用：101.00
报销金额：91.00 (51.00 + 40.00)
自付金额：10.00 (101.00 - 91.00)
```

## 🔐 权限控制流程

### 拦截器工作流程

```
请求 → 拦截器 → 校验流程 → Controller
```

**详细步骤**:

```
1. 从请求头获取 token
   ↓
2. 从 Redis 验证 token
   - key: login:token:{token}
   - 不存在 → 返回 401（登录已过期）
   ↓
3. 获取 userId 并保存到 ThreadLocal
   - UserHolder.save(userId)
   ↓
4. 检查@Permission 注解
   - 无注解 → 放行
   - 有注解 → 继续校验
   ↓
5. 从 Redis 获取用户角色
   - key: login:role:{userId}
   ↓
6. 校验角色匹配
   - 匹配 → 放行
   - 不匹配 → 返回 403（无权限）
   ↓
7. 执行 Controller 方法
   ↓
8. afterCompletion 清理 ThreadLocal
   - UserHolder.remove()
```

### 权限注解使用

**类级别** (整个 Controller 需要特定角色):
```java
@RestController
@RequestMapping("/fee")
@Permission(Role.HOSPITAL)  // 只有医院可以访问
public class FeeController {
    // ...
}
```

**方法级别** (特定方法需要特定角色):
```java
@RestController
@RequestMapping("/visit")
@Permission(Role.HOSPITAL)  // 默认医院角色
public class VisitController {
    
    @GetMapping("/my/list")
    @Permission(Role.PATIENT)  // 这个方法只有患者可以访问
    public Result myList() {
        // ...
    }
}
```

## 🔧 关键技术点

### 1. 分布式锁使用

**场景**: 防止并发操作导致数据不一致

**标准代码**:
```java
String lockKey = "lock:business:{id}";
RLock lock = redissonClient.getLock(lockKey);
try {
    // 尝试获取锁，等待 10 秒，不自动释放
    if (!lock.tryLock(10, -1, TimeUnit.SECONDS)) {
        return Result.fail("操作正在进行中，请勿重复提交");
    }
    // 执行核心业务逻辑
    return executeBusinessWithTransaction();
    
} catch (InterruptedException e) {
    Thread.currentThread().interrupt();
    log.error("操作被中断", e);
    return Result.fail("操作被中断");
    
} finally {
    if (lock.isHeldByCurrentThread()) {
        lock.unlock();
    }
}
```

### 2. 幂等性控制

**场景**: 防止重复提交

**实现方式**:
```java
// 1. 先查数据库
if (existsInDatabase()) {
    setRedisFlag();
    return Result.fail("重复提交");
}

// 2. 业务逻辑

// 3. 设置幂等性标识
redissonClient.getBucket("idempotent:xxx").set("1", 3600, TimeUnit.SECONDS);
```

### 3. 金额精度控制

**场景**: 报销金额计算

**正确做法**:
```java
// 先累加再四舍五入（避免累积误差）
BigDecimal totalRaw = BigDecimal.ZERO;
for (Fee fee : feeList) {
    totalRaw = totalRaw.add(fee.getTotal().multiply(rate));
}
BigDecimal total = totalRaw.setScale(2, RoundingMode.HALF_UP);
```

### 4. 事务管理

**场景**: 保证数据一致性

**标准用法**:
```java
@Transactional(rollbackFor = Exception.class)

    // 多个数据库操作

}
```

### 5. 缓存使用

**场景**: 提高查询性能

**标准用法**:
```java
// 1. 读缓存
String cacheJson = redisTemplate.opsForValue().get(cacheKey);
```
